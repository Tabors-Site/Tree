Spatial Extension Scoping: The Plan
The idea: metadata.extensions.blocked[] on any node. Inherits from parent to child, same as tool blocking. Block an extension at a tree root and none of its capabilities exist anywhere in that tree.

What gets blocked:

Tools from that extension don't appear in AI tool resolution
Hooks from that extension don't fire at that node
Modes from that extension don't resolve at that node
enrichContext from that extension skips that node
setExtMeta from that extension is rejected at that node
What stays: The extension is still installed on the land. Its routes exist. Other trees can use it. It's spatial, not global.

Build order:

isExtensionBlockedAtNode(extName, nodeId) utility (walks parents)
Tool resolution filter (extends existing chain)
Hook filtering (skip blocked handlers)
Mode resolution filter
Metadata write guard
CLI: ext-block, ext-allow
API: POST /node/:nodeId/extensions
No breaking changes. No schema migrations. The infrastructure is already there. This is connecting the last wires.
