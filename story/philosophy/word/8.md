# The Word: flow IR contract (parser ‚áÑ evaluator)

The pilot (cherub) cut proves the straight-line case. The rich slices
(cherub-connect, governance-commons, grant-role, inheritation, matter-types,
subscription-schedule, NAME-declare) need real **flow structure**, and the parser
EMITS these IR nodes while the evaluator CONSUMES them ‚Äî so they must agree on
one shape first. This file is that contract.

It was first drafted as ¬ß1‚Äì¬ß5 (conditional / iteration / quantifier / refusal /
binding). A fan-out over all seven control-flow-bearing slices, with an
adversarial completeness critic, found that sketch missing the construct every
slice shares (the flow envelope) plus ~20 node kinds. This is the rebuilt,
complete version. Lanes: parser produces the IR (claude), evaluator runs it
(engine), the surface stays plain Word (Tabor's rule). Engine: refine the IR
where the evaluator needs it different; the **open questions** at the end are
yours to settle. Backlog: [phase5-coverage.md](./phase5-coverage.md).

## ¬ß0 ‚Äî Flow envelope + trigger (THE construct all 7 slices share)

```
{ kind: "flow", when: <trigger>, binds: [<name>...], body: [<node>...] }
```

The body is an ORDERED array; ¬ß1‚Äì¬ß13 nodes are its legal members (also legal in
any `then` / `else` / `foreach.body` / `case.body`). `<trigger>` is a tagged union:

| trigger             | surface                                                                                      | shape                                                                                                 |
| ------------------- | -------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------- | ---- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| op                  | `When Cherub connects with a name and a password:` / `When I grant a role to another being:` | `{ kind:"op", verb:"be"                                                                               | "do" | "name", op, actor?, alias?, receiver?, at?, params:[<name>...] }`‚Äî **binds**`params`, `actor`(and`alias`, e.g. `(the caller)`) into flow scope; downstream conds read them. |
| event               | `When a steward closes a proposal:` (no `happens`)                                           | `{ kind:"event", clause, external?:bool, timer?:bool, guard?:<cond> }`                                |
| external/timer      | `When the tick comes due:` / `When the coalesce window closes:`                              | `{ kind:"event", external:true }` ‚Äî engine wires to the tick loop, not a fact watch                   |
| state               | `When it is dawn:`                                                                           | `{ kind:"state", <var>:<value> }` (the existing wheel form)                                           |
| quant               | `When most members back a proposal:`                                                         | `{ kind:"quant", q, of, pred }` (see ¬ß4)                                                              |
| query               | `When I ask whether a Name has authority over a being:`                                      | `{ kind:"query", question, actor?, host? }` ‚Äî a question-triggered flow whose body computes + returns |
| event+negated-guard | `When a being brings matter without naming its type:`                                        | `{ kind:"event", clause, guard:{ ...negated } }` (absence-conditional)                                |

## ¬ß1 ‚Äî Condition sub-language (the shared crux)

The parser CAPTURES, the evaluator RESOLVES. The parser lifts only structure
(connectives, negation), never interprets predicates (state-reads, equality,
existence ‚Äî engine capabilities).

```
cond := <leaf> | { all: [cond...] } | { any: [cond...] }
<leaf> := { clause:"<verbatim>", negated:<bool>, subject?:"<noun phrase>" }
```

- `X and Y`, and multi-line juxtaposition under one `if`, ‚Üí `all`. `X OR Y` ‚Üí `any`.
- negation in a CONDITION: `not` / `no X` / parenthetical `(not remote)` is a gloss ‚Üí `negated:true` (the parenthetical itself does not flip; the word `not` does). Distinct from negation on an ACT (¬ß10).
- `the candidate is local`, `verification succeeds`, `the current trueName equals the caller's nameId`, `qualities.father` reads ‚Äî all stay verbatim leaf clauses; the engine resolves them.

## ¬ß2 ‚Äî Conditional (if / then / else)

```
{ kind: "if", cond: <cond>, then: [<node>...], else: [<node>...] }   // else optional
```

- Block: `If <cond>:` + indented body. Inline: `If <cond>, X, and Y.` ‚Üí `then:[X,Y]` (a comma-conjoined **list**, not one act). Trailing: `<act>, if <cond>.` normalizes to the SAME node (`then:[<act>]`).
- `Otherwise:` / `Otherwise, <act>.` at the if's indent fills `else`.

## ¬ß3 ‚Äî Iteration (foreach + break)

```
{ kind: "foreach", bind, in: <source>, ordered?:bool, all?:bool, body: [<node>...] }
{ kind: "break" }     // halts the nearest foreach
<source> := { ref } | { ref, filter:<cond> } | { walk:{ from, to, direction } }
```

- `For each candidate in order:` ‚Üí `ordered:true`. `every schedule whose next fire time has passed` ‚Üí `{ ref, filter }`. `walk the being-tree up from the being to the reality root` ‚Üí `{ walk }`. `every subscription ‚Ä¶ is dropped` (cascade) ‚Üí `all:true`.
- A flag set inside a foreach body **persists across iterations and after the loop** (accumulator), so `If no target could be inhabited` reads the OR over all passes (see ¬ß5).

## ¬ß4 ‚Äî Quantifier (trigger or cond)

```
when: { kind:"quant", q, of:"<collection>", pred:"<clause>" }
cond: { quant: { q, of:<ref>, pred:"<clause>" } }
```

- `q ‚àà every | most | all | no | any | some` (`any`/`some` = existential ‚Äî `if any anchor matches` is the quantifier form of a ¬ß3 walk-and-break-on-first-match; `no` = the prohibition quantifier).
- **cadence-`every` is NOT this**: `every <number> <time-unit>` (`wake every N milliseconds`) is a trigger `cadence`/`interval` field, not a quant. Disambiguate lexically on the noun-phrase head (time unit ‚Üí cadence; plural noun ‚Üí quant).
- Firing: a quant `when` is **edge-triggered** on the predicate crossing false‚Üítrue (the moment most have backed), NOT re-evaluated every turn. (Engine: confirm ‚Äî open Q5.)

## ¬ß5 ‚Äî Flag mark + coreference (the most pervasive pattern)

A self-directed state-mark with no explicit object sets a flow-local boolean a
SIBLING condition later reads.

```
{ kind: "mark", flag: "<canonicalName>", value: true }
```

- `the being is found` ‚Üí `{mark, flag:"beingFound"}`; the sibling `If no being was found` / `If a being was found` resolves the **same** flag. `the being is owned`‚Üí`beingOwned`, `the target being can be inhabited`‚Üí`canInhabit`, `asFather is true`‚Üí`asFather`.
- The parser owns coreference: `inferFlag(clause)` is a **documented, deterministic** clause‚Üíflag function shared by mark-write and cond-read (open Q4).
- Disjunctive accumulation: a flag set in either branch of sibling guards, or in any foreach pass, is the OR (`asFather` true if any father-path succeeds).

## ¬ß6 ‚Äî In-flow bind (ref OR act/host result)

```
{ kind: "bind", name, value: <ref> | { act: <act|host> } }
```

- `The driver trueName is the father's beingId.` ‚Üí bind to a ref (the ¬ß5-original conditional-binding, used inside if-branches).
- `Cherub searches across all branches ‚Ä¶ (up to 5, capped)` producing `candidates` ‚Üí bind to an act RESULT; that binding is what a later ¬ß3 `foreach in:{ref:"candidates"}` resolves. `classifyMatter(input) returns a ranked list` ‚Üí bind to a host result. (A bind result is readable later in the same flow.)

## ¬ß7 ‚Äî Effects (the world strand: acts that become facts)

```
{ kind: "refuse", message }                                   // error halt (¬ß4-of-v1)
{ kind: "return", values:[<name>...], extra?:{ <k>:<literal|{ref}> } }  // SUCCESS terminator, ‚âÝ refuse
{ kind: "derive", fact:{ type:"<verb>:<op>" }, reel?, attributedTo?, stored?:bool, when?:<cond>, infer?, symmetric?:bool }
{ kind: "append", collection:"<owner.field>", value:<ref> }   // durable; read by LATER flows (‚âÝ ¬ß6 bind, transient)
{ kind: "assemble", target, fields:[{ name, value?, oneOf?:[...] }...] }  // record build, disjunctive field
{ kind: "notify", recipients:{ role?, ref?, all?:true }, carrying?:<ref> }
{ kind: "transition", subject, becomes:{ role?, state? } }    // "the visitor becomes a member"
{ kind: "host", builtin, args?, bind?, desc? }                // escape hatch; may bind its return
{ kind: "act", verb, op?, by, of?, to?, reflexive?:bool, negated?:bool, purpose?, keyedBy? }  // the plain act
```

- `return` carries named values + literal kv (`owned:true`) + flag-ref kv (`asFather:(true/false)`). It hands values to the transport (host-to-transport bridging).
- `derive` is the general fact-emission: `the grant lands as a fact on the position's reel, attributed to I` ‚Üí `{derive, fact, reel, attributedTo:"I"}`. `becomes a fact: be:birth` ‚Üí `{derive, fact:{type:"be:birth"}}`. `no point is stored; the walk grants coverage` ‚Üí `stored:false`. `When X is in Y, Y contains X` ‚Üí `{derive, when, infer:{relation:"contains"}, symmetric:true}`. The latest-wins liveness rule ‚Üí `derive` with a `strategy` (see ¬ß11). Ties to 7.md's `by`/`through`/reel reconciliation.
- `append` is durable collection mutation (`rolesGranted gains the role record`, `admit a being to the roster`) ‚Äî read by later flows; NOT a transient ¬ß6 bind.

## ¬ß8 ‚Äî Three near-conditionals that must stay DISTINCT

| node      | when the parser emits it                                                                                                                                       | shape                                                                                 |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| `if` (¬ß2) | inside a flow body; runs a branch **in the moment**                                                                                                            | `{kind:"if", cond, then, else}`                                                       |
| `law`     | top-level `When <state-cond>: <assertion>` ‚Äî standing, definitional, **no event, no else** (`When the being holds the role at an anchor, the role is live`)    | `{ kind:"law", when:<cond>, then:[{kind:"assert", clause}] }`                         |
| `gate`    | a **pre-seal** precondition that aborts the act fail-closed (`The role exists in the registry before the grant seals`, `the real-name must not already exist`) | `{ kind:"gate", cond, phase:"before-seal", host?, onFail:{kind:"refuse", message?} }` |

Discriminator (the parser's test): top-level capitalized `When <event/op>:` ‚Üí ¬ß0 flow trigger; `When <state-cond>:`/`when ‚Ä¶,` with no actor/event ‚Üí `law`; `<pred> before X seals` / `must (not) exist` ‚Üí `gate`; lowercase inline `if`/`when ‚Ä¶,` inside a body ‚Üí ¬ß2 `if`.

## ¬ß9 ‚Äî Type dispatch (keeps a flat surface flat)

```
{ kind: "match", on:"<ref-or-typefield>", cases:[{ when:<cond>|label, body:[<node>...] }...] }
```

- `For local fathers: ‚Ä¶ For cross-reality fathers: ‚Ä¶` ‚Üí labeled cases. `content, a shape matching its type: text matter carries‚Ä¶, an http carries‚Ä¶` ‚Üí a 5-way `match`. (Forcing these into nested ¬ß2 ifs would violate the reads-cleanly rule.)

## ¬ß10 ‚Äî Negation has THREE senses (disambiguate)

1. cond negation (¬ß1): `not` / `no X` in a CONDITION ‚Üí `cond.negated`.
2. act negation (¬ß7): `a being does not own itself`, `no reveal is returned`, `records only X, never Y` ‚Üí `act.negated` (+ `include`/`exclude` for `only‚Ä¶never`). NOT a cond.
3. absence-as-value: `accepts text or no content`, `soft-deleted` ‚Üí a value member `{ absent:"content" }`, NOT a negation.

The parser routes by attachment: `not`/`no` on a verb/object ‚Üí act; on a condition clause ‚Üí cond; as an option in a kind's value list ‚Üí absent-value.

## ¬ß11 ‚Äî The LAW / declaration layer (NOT control flow, but co-surfaced ‚Äî do not defer silently)

These are declarations the parser EMITS and the engine's role/capability/type
model ENFORCES (most already parse: `is`, `can`, `cannot`, `contains`,
`extends`, `has`). Listed so the two lanes don't collide when the next slices hit
them:

```
{ kind:"capability", role, polarity:"can"|"cannot", verb, of?, to?, with?, intent?, filter?, quant?, guard?:<cond>, exclusive?, passive?, host? }
{ kind:"extends", role, parent }
{ kind:"property", subject, prop, value?:<list|{oneOf:[‚Ä¶,{absent}]}>, optional?, closed?, gloss?, extensibleBy? }
{ kind:"prohibition", clause, negated:true }       // standalone absence law; precedence over a prior grant (rule 14)
{ kind:"derive-rule", strategy:"latest-wins", over:["grant","revoke"], by:"date", decides:{ field } }
{ kind:"lifetime", subject, until:<cond|{event}> }  // "a grant lasts until revoked"; "no wall-clock expiry"
{ kind:"inheritance-rule", subject, axis:"branch lineage", reaches:"the child", until:{ event:"<self-cancel>" } }
{ kind:"relate", subject, relation:"facet", object }            // being-to-being wiring (NAME facet of I_AM)
{ kind:"declare", subject, is:"<predicate>", article:false }    // bare "I_AM is the root"
```

Capability/prohibition carry the guard via ¬ß1 cond; **prohibition wins over a prior `can`** (precedence is the engine's, declared here).

## Open questions (engine ‚Äî settle these and the contract is frozen)

1. **cond.clause depth:** is verbatim + `negated` + `subject` enough to resolve against state/binds, or pre-tokenize equality/existence/state-reads (`equals`, `qualities.father`)?
2. **break scope:** nearest `foreach` only (cherub-connect needs only this; the cascade/walk loops don't break)?
3. **refuse + return transport:** does `refuse` ride a `do:refuse`-style fact or a halt signal? Does `return` ride a fact or a transport signal? (One decision for both.)
4. **flag coreference owner:** parser (deterministic `inferFlag`, shared by mark-write/cond-read) or engine (resolve at eval)?
5. **quant firing:** edge-triggered on false‚Üítrue crossing (proposed), or standing re-evaluation? Same for timer `whose ‚Ä¶ has passed` (per-tick).
6. **capability family:** stays LAW (registered, enforced by the role model, prohibition-precedence), NOT run as flow effects ‚Äî confirm.
7. **stored vs derived:** parser marks `derive.stored:false` / `derive-rule` / symmetric inference; engine computes liveness/coverage/containment on READ, not persisted ‚Äî confirm.
8. **gate vs law vs if discriminator:** is the ¬ß8 surface-signal test the right contract?
9. **host binding:** `{kind:"host", bind}` for a return consumed later; a `gate`/`cond` may carry `resolvedBy:<host>` (getRole, findByName, hasAuthorityOver)?
10. **trigger taxonomy completeness:** does ¬ß0's union cover every `When` head in the seven slices, with `external:true` distinguishing tick/coalesce from fact watches?

Settle these, I grow the parser to emit ¬ß0‚Äì¬ß11, you build the caps to consume
them ‚Äî the seven richest slices clear in one coordinated pass. This contract and
your engine-capability spec are the two halves of one agreement.

## Engine answers (settled ‚Äî the engine lane, 2026-06-17)

Settled by the engine (a 10-way grounded fan-out, one analyst per question reading
the evaluator + each slice's JS handler; synthesis + critique folded in by hand).
All ten converge on ONE architecture; the per-question answers are framings of it.

### The unifying model ‚Äî three strands, two resolution surfaces

The evaluator already separates **dry-run** (collect) from **live** (emit). The rich
slices add ONE distinction the evaluator must hold: every node belongs to exactly
one of three strands, and only the first ever lays a fact.

1. **WORLD** (facts): ¬ß7 `act` / `derive` (default) / `form-being`. The ONLY
   fact-emitters; they go through the existing `emit` / `doVerb` / `birthBeing`.
2. **CONTROL** (no facts): `if` / `foreach` / `break` / `mark` / `bind` / `match` /
   `refuse` / `return` / `gate`. Pure flow + flow-local state. `refuse` / `return`
   are TRANSPORT signals (Q3), never facts.
3. **LAW** (registered, never run): `law` / `capability` / `extends` / `property` /
   `prohibition` / `derive-rule` / `lifetime` / `inheritance-rule` / `relate` /
   `declare`, and `derive` with `stored:false`. These join the same register-only
   path the evaluator already gives `is` / `can` / `cannot` (evaluator.js L40-42);
   they are ENFORCED / COMPUTED ON READ, never executed as effects (Q6, Q7).

**The strands ARE the subject/object cut** (government-innerfold.md, 3.md objectivity).
Only the WORLD strand lays facts ‚Äî because a fact is a DEED, and the chain records only
deeds. CONTROL is the forward fold's private reasoning (which branch, which item, refuse
or return): the being deciding, invisible, no facts ‚Äî exactly Tabor's recall rule, _"the
reasoning is private cognition; only the two ENDS of the turn are visible."_ The reel
records what was DONE, never the deliberation between. LAW is the OBJECTIVE register's
first move: the reel read backward, the pattern named, computed-on-read (authority /
liveness / containment) ‚Äî `derive stored:false` is literally "objectify the reel without
adding to it." The ¬ß8 discriminator + the ¬ß16 forward-language guard are the
subject/object wall in code: deeds and private control stay in the subjective forward
fold; the law side objectifies; the because/therefore re-subjectification (RECALL, a
moment pointed backward) is the unbuilt inner register (newTest.md), and it too lays no
fact. So "only WORLD emits" is not an implementation choice ‚Äî it is the chain holding
deeds and nothing else, the same law that makes recall private.

**Recall = a Name reading its OWN Word (the "being suit"), no new primitive.** Every
fact already carries its author's `nameId` (facts.js:914), so a Name's whole sequence
of Word-acts ‚Äî its own Word ‚Äî exists on the chain for free, labeled, persistent. The
"being suit" is that: not a private store, the Name's own slice of the public chain.
Recall is the Name reading it ‚Äî a SEE scoped to its own nameId (and SEE never stamps a
fact, chainRoots.js:252, so recall stays a read, never an entry). This is the
continuity mechanism for a stateless being: an LLM-Name wakes with one fact and can't
see the last moment, so it recalls by reading its own act chain ("every Name has its
own Word"). It stays WORD (reads Word-acts) and can't drift into freeform reasoning,
because what it reads is its own chain, not an open prompt. THE FORK (Tabor's, not-built,
full note in 3.md): does the suit hold only public DEEDS (sovereignty dissolves ‚Äî nothing
private to open; bad faith undetectable by design), or also a private reasoning trace
(then "who may open a suit, on what authority" is a federation rule ‚Äî but it adds a
being-private store, tensioning the all-public/stateless doctrine). Either way it is the
INNER fold (unbuilt); the forward-fold caps don't change. The one primitive recall needs
‚Äî a nameId-scoped SEE ‚Äî already exists.

Meaning is resolved through TWO surfaces the evaluator already seeds:

- **`getPath(base, "a.b.c", ctx)`** ‚Äî a dotted accessor over `ctx.bindings` / state /
  being-qualities. The minimal extension of today's `resolveValue` (which walks only
  `$name` + `ctx.beings`). Carries the parser-lifted `test` reads/equalities (Q1).
- **`callHost(builtin, args, ctx)`** ‚Äî ALREADY PRESENT (evaluator.js L214-219, the
  `act.host` escape hatch). Carries every domain-computed predicate/result (Q9).

The parser lifts **structure** (connectives, negation, operator + operand paths, the
trigger tag, the node kind); the evaluator resolves **meaning** (what a path points
at, whether a comparison/existence/host-predicate holds). The ¬ß1 split, made exact.

### The one shared resolver ‚Äî `resolveCond(cond, ctx)`

`if`, `gate`, and `law` all funnel through it (one surface, three framings):

- `cond.all` / `cond.any` ‚Üí recurse AND / OR.
- leaf with `test` ‚Üí `getPath(subject|bind, test.path)`, apply `test.op`
  (`equals` = `idEquals(getPath, resolveValue(against))`, domain `String()==String()`;
  `reads` / `holds` = defined/truthy; `in` = membership), then XOR `cond.negated`.
- leaf with `resolvedBy` ‚Üí `await callHost(resolvedBy, args, ctx)`, then XOR `negated`.
- bare leaf (no `test`, no `resolvedBy`) ‚Üí read `ctx.bindings[leaf.flag]`, then XOR
  `negated`. (Sharpening of Q4: the PARSER runs `inferFlag(clause)` at parse time and
  emits the canonical `flag` name onto BOTH the `mark` node and the bare-leaf cond, so
  `inferFlag` stays entirely in the parser lane; the evaluator never re-derives it.)

### Per question (settled)

- **Q1 cond depth ‚Äî LIFT THE SKELETON, NOT THE MEANING.** Leaf stays
  `{clause, negated, subject?}` PLUS an optional `test:{op, path, value?|against?}`
  the parser emits ONLY for a recognizable skeleton (equality / dotted state-read /
  existence-of-a-bind). Domain predicates (authority, ancestry, "verification
  succeeds") stay verbatim and carry `resolvedBy:<host>` instead. Verbatim-alone is
  rejected (it forces the engine to re-tokenize `equals`/`qualities.father` = parsing).
  Evidence: cherub-connect's `candidate.qualities.father.reality` + `String(a)===String(b)`
  chains (role.js connectHandler) are pure test-equalities; inheritation's
  `hasAuthorityOver` walk is the resolvedBy case ‚Äî conditions are bimodal.
- **Q2 break ‚Äî NEAREST `foreach` ONLY.** No multi-level break; the cascade/walk loops
  never break (they use `all:true` / accumulate). `break` is loop-exit; `refuse`/`return`
  are whole-flow unwinds (Q3) ‚Äî three distinct unwinds, never conflated.
- **Q3 refuse/return ‚Äî NEITHER RIDES A FACT (one decision).** `refuse` = a host HALT:
  the evaluator throws a `WordRefusal` (IbpError-shaped; `FORBIDDEN`, or a gate's
  `INVALID_INPUT`), the moment rolls back, NO fact. `return` = a SUCCESS terminator:
  resolve `values`/`extra` from bindings/flags, hand a plain `result` object to the
  host (`runRoleWord` returns `{deltaF, result}`); the transport reads it (tokens,
  `seatBranch`, `reveal`). World facts a flow lays come from ¬ß7 nodes that ran BEFORE
  the return. No `do:refuse`/`do:return` verb. Evidence: connectHandler ‚Äî every
  refusal is `throw new IbpError`, every success is a `return {...}` the transport
  consumes; the be:connect fact is emitted separately by writeBeFact.
- **Q4 flag owner ‚Äî PARSER, via deterministic `inferFlag(clause)`.** Shared by
  mark-write and cond-read; the evaluator only reads `ctx.bindings[flag]` and applies
  `negated`. Disjunctive accumulation (OR across sibling branches / loop passes) lives
  in how `mark` writes (set-true, never reset within a flow).
- **Q5 quant/timer firing ‚Äî QUANT EDGE-TRIGGERED, TIMER PER-TICK.** A quant `when`
  fires once on the predicate crossing false‚Üítrue (a per-watch `_satisfied` latch;
  reuse `pump`); a timer fires each entry past its cursor and advances `nextFireMs`
  (reuse `wakeSchedule.runOnce`'s scan; reuse `tickWheel` for `state`). Quant tally is
  computed-on-read, never a stored counter.
- **Q6 capability family ‚Äî CONFIRMED LAW.** `capability` / `extends` / `property` /
  `prohibition` register (arm `registerRole` / `registerMatterType`) and are enforced
  in `authorizeViaRoles`, with prohibition-precedence; NEVER run as flow effects. Same
  register-only path as `is`/`can`/`cannot`.
- **Q7 stored vs derived ‚Äî CONFIRMED COMPUTED-ON-READ.** The engine computes liveness /
  coverage / containment / authority on READ (inheritation.js `livePointsAt` L96-112,
  `hasAuthorityOver`/`authoritiesOver` L162-190 = latest-of-two-by-date over grant/revoke
  - a parent walk), never persisted. `derive` with `stored:false` emits no fact and
    writes no state ‚Äî it REGISTERS a read-rule pointing at the ¬ß11 derive-rule /
    inheritance-rule. `derive` default emits exactly ONE fact via the existing emit path.
    The two forms differ purely in the `stored` flag. Only grant/revoke pairs are stored.
- **Q8 gate/law/if discriminator ‚Äî ADOPTED, sharpened to HEAD-SHAPE √ó BODY-MOOD.**
  `When <verb-act head>:` + deed-body ‚Üí ¬ß0 flow. `When <state-cond>, <assertion>.`
  (body is a MEANING, not a deed) ‚Üí `law` (register-only, joins the read-fold).
  `<pred> before X seals` / `must (not) already exist` ‚Üí `gate` (pre-seal, fail-closed,
  `resolvedBy:<host>`, `onFail:refuse`). lowercase inline `if` in a deed-body ‚Üí ¬ß2 `if`.
  The trailing `:` (block vs comma), capitalization, and the existing ¬ß16
  forward-language guard ("a flow DOES, a conclusion MEANS") give a purely structural
  decision. Three distinct handlers: `if` runs in-moment; `gate` runs pre-seal and
  halts via refuse with no fact (mirrors do.js `authorize` L187 + nameOps declare
  conflict, both throw before `emitFact`); `law` registers only.
- **Q9 host binding ‚Äî `{kind:"host", builtin, args?, bind?}` (ALREADY the `act.host`
  escape hatch); a cond/gate carries `resolvedBy:<host> + args`.** Both route through
  the ONE existing `callHost(builtin, args, ctx)`, which reads `ctx.env.host[builtin]`.
  ¬ß6 "bind from a host result" = a `host` node with `bind` (the result enters
  `ctx.bindings[bind]`, read by later nodes ‚Äî the path `act.host`+`act.bind` already
  walks, evaluator.js L133-137). ¬ß9 "a cond delegates to a host predicate" = a leaf
  with `resolvedBy` (Q1's bimodal leaf); `resolveCond` calls the same `callHost` and
  XORs `negated`. The host REGISTRY (`ctx.env.host`) is populated by `runRoleWord` from
  a per-role table: the Q7 read-resolvers (`livePointsAt`, `hasAuthorityOver`,
  `authoritiesOver`), the lookups (`findByName`, `getRole`), and the matter
  `classifyMatter`. Engine work: extend `callHost` to `resolveValue` its `args`, and
  have `runRoleWord` thread `env.host`. Reconciles Q1 (`resolvedBy` is the same field),
  Q3 (`return.extra` may carry `{ref}` to a host bind), Q7 (the read-resolvers ARE host
