## Rung 3 — the `.word` authors the verb-op — DECISION: per-op `.word` (option C)

The fold made the op a Word the runtime READS; rung 3 makes the `.word` the thing the op is WRITTEN
as (the honor-vs-betray gap from the trace: today the op-SET source is still JS — `declareNameOpsToFold`
reads NAME_OPS). **Decided C, not a "can block"**, after reading the other agent's landed work
(portal.word / create-matter.word / grant-role.word / move.word — *portal.word is ground truth, the
others may be partly aspirational*). Every op there is a **per-op `.word`** with one shape:

```
When a being <does X>:
  see <floor-read>(...) as $var          # the floor (crypto/CAS/clock/fold), reached through SEE
  If <cond>, refuse "..." as <kind>       # gates — .word
  do <other-word> with { nested $refs }   # compose — words on words
  Return key: $var, factParams: $enriched # dispatcher stamps from factParams; the word never stamps
```

A "can block" (option A) is a half-measure: op-set in the Word, **logic still JS**. C makes a verb-op
the SAME kind of Word as a do-op → LLM-authorable, sealable, source-agnostic. Cost (accepted): the verb
dispatch (be.js/name.js) must **adopt `_factParams`** (stamp from a returned spec, like do.js) instead
of hardcoded `writeBeFact`/`writeNameFact`; and ~37 per-op `.word`s eventually. The crypto looks
irreducible but becomes a `see` floor-read (like `grant-stamp`/`resolve-birth-spec`).

## The endgame this serves (why it matters)

The Word layer becomes **source-agnostic**: a sealed `.word`, a form input, and a live LLM decode are
the same kind of thing — *ongoing Word composition → continuous evaluation → optional sealing into a
Fact*. The dispatcher owns the stamp, so an LLM-being (like a `.word`) composes freely but physically
cannot violate one-Fact-per-act / caller-attribution / content-addressing / presentism. form-portal
proved the sealed-file source; the LLM loop is the live source; verb-ops-as-Words closes the last
corner an LLM couldn't author. (See llm.md, language.md.)
