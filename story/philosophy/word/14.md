# 14 — The translation layer: the cognition speaks Word

13.md §1 stated it in principle: a cognition receives Word and outputs Word, not JSON mimicking MCP. This file makes it concrete against the cognition path we have **now** — what it looks like, what it needs, what it retires — and ends with a phased plan. It's grounded in the code, because the plumbing is mostly already there; the barrier is one format.

## §0 — Where we are (the current path)

A being with `llm` cognition (`beingCognition` → `"llm"`, materials/being/identity/lookups.js:119) takes a turn as a **moment** (`runMoment`, present/moment.js:65) in four beats: assign → fold → momentum → stamped. The cognition runs in beat 3:

- **Input.** `buildSystemPromptForRole` (present/cognition/llm/assemble.js) renders a prompt — *"I am ⟨name⟩, ⟨role⟩ at ⟨space⟩. and can: do: […] see: […] summon: […]"* — plus the SEE-resolver blocks (the world face) and, on half/inward, a past-face. Alongside it the LLM is handed **JSON tool schemas** — `do(action, target, args)`, `call(content, …)`, `be(…)` — in OpenAI function-calling form (present/cognition/llm/tools.js, seedDoTool.js / seedCallTool.js / seedBeTool.js).
- **Call.** `runLlmMoment` (present/cognition/llm/llmMoment.js:156) makes one provider call (`callWithFailover`, call.js) and parses the response.
- **Output.** The LLM returns a **tool_call** — `{ function: { name: "do", arguments: JSON.stringify({ action, target, args }) } }` — dispatched by `executeTool` → the handler → `doVerb(target, action, args, { identity, moment })`. Facts land on `moment.deltaF`; the seal commits Act + ΔF atomically.
- **Outcome.** One of `{ kind: "act" } | { kind: "see" } | { kind: "failure" }` gates the seal.

Beside it, the Word runtime **already exists** — just not as a cognition. `runRoleWord(ir, { moment, history, trigger, bindings, beings, env })` (present/word/roleWordRegistry.js:280) parses a `.word` to IR and `evaluate`s it (evaluator.js), emitting facts onto **the same `moment.deltaF`**. It's how cherub.word, create-matter.word, and set-world-signal run today. The Word verbs (see/do/call/be/recall) map to the seed verbs.

So the two paths **already share the plumbing** — the moment, the `deltaF`, the seal, the outcome shape, the scheduler. The only thing that differs is the cognition's **output format**: a JSON tool_call (parsed by `executeTool`) versus Word text (parsed by the word parser). The barrier is the format, not the machinery.

One more grounding: the **human** cognition already speaks Word. The book view types Word live (the scribe — typeIntoBook). So the human types Word into the book; the llm types JSON into a tool_call. The translation layer brings the llm to where the human already stands.

## §1 — What it looks like

The Word-native moment, beat for beat, is the current moment with **two swaps and one addition** — everything else reused:

1. **Input — the face and the vocabulary, as Word.** The cognition receives the **face folded into Word** (the book render — present tense for SEE, past tense on recall; what the book view already produces) and its **vocabulary**: the words it may speak — its granted `do` ops, `see` reads, `call` targets — rendered as **Word grammar**, not JSON schemas. The "can: do:/see:/summon:" list assemble.js already builds *becomes* the vocabulary; the cognition chooses among declared words. This is the **guide** of 13.md §1 — in-vocabulary by construction.
2. **Call — the cognition emits Word.** One provider call (`callWithFailover`, reused), but the response is **Word text** in the role's grammar: `do create-space .config.` / `call @tabor "the grid is full."` / a whole **flow** (`if …`, `foreach …`, `mark …`, `refuse with "…"`, `return …`) — the roleflow the parser already reads.
3. **Parse + run — the word parser, then `runRoleWord`.** The output is parsed by the **word parser** (parser.js) to IR and run through **`runRoleWord(ir, { moment, … })`** — the *same* executor as cherub.word, onto the *same* `moment.deltaF`, sealed the *same* way. No `executeTool`, no `seedDoTool`.
4. **The addition — translate on miss, fail closed.** If the output doesn't parse, a **second call** (the Pentecost interpreter) rewrites *what it meant* into valid Word; re-parse. If it still won't parse, the moment **fails closed** — `{ kind: "failure" }`, refuse, **never run a half-parsed Word.** (Because the cognition's Word now *executes*, malformed Word is more dangerous than malformed JSON, which merely failed to dispatch — so the gate is strict.)

The diff, precisely: **swap** the input render (Word grammar + vocabulary, drop the JSON schemas) and the output parse (word parser → `runRoleWord`, drop the tool_call dispatch); **add** the translate-on-miss + fail-closed gate; **reuse** `runMoment`, the seal, the outcome shape, the scheduler, `callWithFailover`, the book-render face. It is not a new cognition kind — it is how `llm` cognition *works*, the way human cognition already types Word.

## §2 — What it needs (the gaps)

Grounded in the files, smallest-first:

1. **The vocabulary render (the guide).** Extend assemble.js to emit the being's granted words as **Word grammar** (the verbs + the available word-names + their arg shapes), replacing the JSON tool schemas. The data is already there (the canDo/canSee/canSummon lists); the work is rendering it as Word the cognition picks from, not an OpenAI `parameters` block. *Small–medium.*
2. **The output parse wiring.** Point `runLlmMoment`'s response handling at the **word parser → `runRoleWord`** instead of `executeTool`. Parser and executor exist; the work is feeding free cognition output (which may be partial or chatty) through them, and separating the **prose** (rides as `Act.endMessage`, like today) from the **Word** (executes). *Medium.*
3. **The translate-on-miss call.** A second `callWithFailover` pass: given the malformed output + the grammar + the vocabulary, return valid Word. First pass creative, second corrective. *Small, new.*
4. **Fail-closed gate.** On repeated parse failure → `{ kind: "failure" }`; never partially execute. The outcome shape already carries `failure`; the work is wiring invalid-Word to it and ensuring no IR runs unless it parsed whole. *Small.*
5. **Flow per moment (roleflow → flow).** Today one moment = one tool_call = one act. A Word flow is **many acts + control** in one moment — and the seal already commits a multi-fact `deltaF`, so this *works*, but it changes the grain: a moment can now lay a whole flow. Decide whether to keep one-flow-per-moment (clean) or let the flow summon mid-moment. *Design call, not code.*

The input face (the book render), the runtime (`runRoleWord`), and the moment/seal/scheduler are **already built** — not in the gap list. That's why this is mostly a swap, not a build.

## §3 — What it buys

- **Tokens.** The JSON tool schemas (three tools with `parameters` blocks and descriptions, every call) and the `arguments: JSON.stringify(...)` wrapping collapse to the Word grammar (terser) plus the vocabulary (the fold both ends share). The cognition says `do create-space .config.`, not `{"name":"do","arguments":"{\"action\":\"create-space\",\"target\":\".config\"}"}`. Measure the delta on a real role; it makes the §1 promise checkable.
- **One path.** The cognition's Word lands on `moment.deltaF` through `runRoleWord` — the **same** path as every other word. `executeTool` / `seedDoTool` / `seedCallTool` / `seedBeTool` (the JSON envelope) **retire**. Per 13.md §6, that envelope is the *last registry-shaped thing in the loop* — the cognition's private Map of how to phrase an act — and this deletes it, the way the fold deleted the five Maps.
- **Capability.** A moment can emit a **flow**, not one call — conditionals, loops, refusals, several acts — the role's own grammar, run by the evaluator. Richer than one tool_call per wake.
- **The shape is Pentecost, not Babel** (13.md §6). The vocabulary is the shared dictionary; the translate-on-miss is the interpreter that lets the cognition be understood without being forced into a rigid schema; fail-closed is the safety. The answer to unreliable output is *more Pentecost* (freer Word + translation), not *more Babel* (stricter JSON).

## §4 — The plan

Phased so each step is verifiable and the JSON path stays live until the Word path is proven.

1. **Render the vocabulary + face as Word** (input). Extend assemble.js to emit Word grammar + the granted words; gate behind a per-role flag (`word-native`) so the JSON path is untouched for others. Verify: a role's prompt renders as readable Word with its true vocabulary.
2. **Parse the cognition's Word → `runRoleWord`** (output). In `runLlmMoment`, when the role is `word-native`, route the response through the word parser → `runRoleWord`, prose split off as `endMessage`. Verify: a word-native being lays the right facts on `deltaF` from a Word answer, sealed like today.
3. **Translate-on-miss + fail-closed** (safety). Add the repair call and the strict gate. Verify: malformed output is repaired or refused, never partially run.
4. **Measure + widen.** Token delta vs the JSON path on a real loop; widen `word-native` from one role to the default.
5. **Retire the JSON envelope.** Once the default is Word, delete `executeTool` / the three seed tools / their schemas. The cognition seam now reads the fold like everything else.

## Open questions

- **Partial parse.** How forgiving is the parser on chatty/incomplete output before the translate pass — strict (translate often) or lenient (risk mis-parse)? Lean strict + fail-closed; the translate pass is cheap and the run is real.
- **Vocabulary completeness.** Does the cognition see *all* its words (the full fold) or a curated subset? Too many is tokens back; too few is capability lost. The fold knows the true set — render that.
- **Orientation.** forward/half/inward (the past-face) is only partly built (forward live). Word-in/word-out doesn't need it, but the face render should honor it when it lands (recall = the past in Word).
- **The other cognitions.** scripted (`role.call`) already runs code — it could *emit* Word too, but needn't. human already types Word (book view). So Word-native is the llm joining a convergence already underway, not a lone new path.
