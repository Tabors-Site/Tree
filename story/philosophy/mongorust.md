The fold/cache, lightest possible, faster-than-Mongo for reads:

The truth is the file. The read-speed comes from a two-layer cache, both derived, both rebuildable from the files:

Layer one — the head snapshot per reel. For each reel, keep the current folded state (the present — what Mongo's document was) as a single small file beside the log: <id>.head (one JSON object, the folded state, plus the seq it's folded through). Reading a being's present = read one tiny file, done, no replay. This is the materialized projection, and it's faster than Mongo because it's a direct file read of exactly the shape you need, no query planner, no BSON decode, no network hop. When a fact appends to <id>.reel, you fold that one new fact into <id>.head and rewrite the head file (it's small). So write = append the fact + update the head. Read = read the head. Both O(1)-ish.

Layer two — an in-memory map, the hot cache. Keep the heads you've touched recently in a plain in-process Map(id → foldedState). Read checks memory first, falls to the .head file, falls to replaying the .reel (cold start / cache miss). This is the wakes-pattern you already use. Memory is the fold-of-the-fold; the .head file is its durable form; the .reel is the truth. Three layers, each rebuildable from the one below, the bottom one being the only truth. This is your own "fold is a cache, never the truth" doctrine made literal in the filesystem.

So the read path, fastest to slowest: in-memory Map → .head file → replay .reel. The write path: append to .reel, fold into the head (memory + .head file). That's the whole storage engine. It's lighter than Mongo because there's no query engine, no schema, no BSON, no connection pool — it's fs.appendFile and JSON.parse. And it's faster for the read you actually do constantly (give me the present of this reel) because that read is now a single small file or a memory hit, not a query.

The honest costs, the things Mongo did for free that you now own — and which one will bite:

The one that bites: cross-reel atomicity. Your composite-op moment lays N facts across N reels (the take-laying-two-facts thing, form-portal, the genesis sequence). Concurrent writes to one reel — solved because a reel is single-writer (a being writes its own chain; the stamper serializes). One append at a time per file, which is the natural grain.
