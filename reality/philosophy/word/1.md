# The Word

The Word is TreeOS's native language: a word-shaped, declarative language that is also the substrate's universal interface. You write what a reality contains, what its inhabitants can see, do, and summon, and how it responds to events. The substrate reads the words and runs them. Same surface for humans and LLMs; same artifact for definition, spawning, modification, and query.

Extension: `.word`. The name is literal, not metaphorical. The language is a structured language of words that does what the Logos does: form is spoken into being through declaration.

> This distills [0.md](./0.md), a long evolving dump. The points below are the evolved end state. What the exploration tried and then moved past is collected under "Evolved away from" at the bottom, so nothing is lost but the body reflects the accurate picture.

## The core move

Today can-see / can-do / can-summon are imperative JavaScript functions embedded in role definitions: imperative islands in a declarative architecture. The Word replaces them with declarations, patterns over the substrate's own structure. Not function calls, not object access. Descriptions of what is.

The decisive commitment, the one that justifies the name and the whole direction, is that the declarations are **word-shaped**: not merely syntactically declarative, but literally constrained natural-language sentences.

```
A dj can see everything in the music-room.
A dj can queue a song to the playlist.
When a listener requests a song, if the requester has good taste, queue the song.
```

Read aloud, that is English. Parsed by the substrate, it is a complete declaration. Both readings are correct.

## Why word-shaped (the reason it is architecturally right)

- **LLMs author natively.** An LLM writes The Word the way it writes anything else. No special syntax to learn, no JavaScript to generate and review. The substrate's organizing layer becomes directly accessible to the cognition that increasingly participates in computing.
- **Humans and LLMs converge on one artifact.** A role a human writes and one an LLM writes are the same kind of thing. The cognition layer records WHO wrote each declaration; the declarations themselves are uniform.
- **Realities self-describe in literal English.** To know what a foreign reality is, read its declarations. They are sentences. Documentation and code converge.
- **Modification becomes conversation.** Proposing a rule change is writing a sentence. The political process of governance and the technical process of declaration happen in the same medium.
- **Translation is meaningful.** A role written in English can be translated to Spanish or Japanese; the substrate operates on the meaning, not the surface form. Realities become multilingual at the architectural level.
- **Accessible.** Anyone who can write a clear sentence can write The Word. The barrier to making substrate applications drops to literacy.

## Primitives are parts of speech

The Word does not need an invented grammar. The substrate's primitives already ARE the categories natural language uses, because both describe reality.

- **Nouns** are the three kinds of thing: person is a being, place is a space, thing is matter.
- **Verbs** are the acts: see (perceive), do (act on things), summon (reach a being), be (exist as), plus name (declare identity).
- **Position** is the space hierarchy (in, inside, contains).
- **Tense:** present is the moment (the current fold), past is the reel (recorded facts). There is no real future tense; TreeOS declares rules, it does not predict.
- **Conditionals** are if / when / while / until, which become flow rules.
- **Adjectives** are qualities and properties of beings, matter, and spaces.
- **Adverbs** are modifiers on how an act is performed (act qualifiers).
- **Pronouns** are variable binding (the requester, they, it), resolved by context.
- **Possessives** are ownership (a Name owns its beings and matter).
- **Plurals and quantifiers** are set operations and quantified queries (all visitors, the next 3 songs).
- **Articles** are definite versus indefinite: queue a song (any match) versus queue the song (a specific referenced one).
> **Status (open).** The core mapping is solid and approved: nouns are being / space / matter, verbs are see / do / summon / be / name. The extended grammar (adjectives, adverbs, pronouns, possessives, plurals, articles) is a candidate set, not yet approved and possibly over-complex; each needs to be checked against real `.word` programs. Default to minimal: add a category only when a program actually needs it, and prefer a property or the escape hatch over a new grammatical construct.
The deep claim: the substrate is not a model of computation, it is a model of reality. Natural language evolved to describe reality. They converge because they do the same job. So The Word borrows natural-language grammar directly, as a constrained, parseable subset of it.

## Code is substrate data

A declaration is data, not opaque code. That one property cascades:

- **Queryable.** "What facts does this act produce?" "What state does it read?" "Can two acts conflict?" become answerable from the declaration without running it.
- **Composable.** Two role declarations compose by the substrate merging declarations, not functions. Roles can extend other roles (`A facilitator extends member.`).
- **Federable.** A declaration serializes (content-addressed), publishes through a Store, and installs in another reality, the same way any resource does. Behavior federates like content.
- **Self-describing (meta-circular).** A reality contains its own definitions in the same notation as everything else. To understand an act, read its declaration.
- **Security is structural.** The language only allows declared substrate operations, so an installed role cannot exceed its declared boundaries. There is no code injection because there is no code to inject.

## It dissolves the see-registry

The first concrete architectural payoff. Today a registry of see-op handlers (JavaScript functions) is looked up at runtime. With The Word, the role's can-see IS the declaration, and the substrate evaluates it directly. This collapses several things at once:

- The see-op handler contract disappears; declarations replace handlers.
- The depSet (which reels were read) is derivable from the patterns, because the patterns say what they query. The fold-scope optimization becomes automatic.
- The permissive-versus-minimal question dissolves: the declaration is the exact upper bound.
- There is nothing to migrate, because the reads live in the role declaration itself.

The substrate gets smaller and more uniform: data drives behavior, behavior is data, all the way down.

## Words define words (generative vocabulary)

A "do" can be many words that become one word. A script is the definition of a word, its etymology, expressed in The Word itself.

```
A fade-out is an act that gradually decreases the volume to 0 over a duration.
A gentle-end performs a fade-out over 10 seconds, then removes the current-song, then announces "the music has ended."
A dj can perform a gentle-end.
```

This makes the language self-extending and fractal:

- The base vocabulary is tiny (the primitive verbs plus a few composition operators). Everything else is words defined in terms of other words, grounding out in primitives. Primitives are not architecturally special; they are just the words whose definitions are built into the substrate.
- Vocabulary grows organically. Communities develop dialects; useful words spread through federation; the same word can mean different things in different scopes (word competition, resolved by scoping), and that is recorded.
- Words carry provenance. The cognition layer records who defined each word, so a reality's vocabulary has attribution: human-defined, LLM-defined, or script-defined. The full cognition-transparency story (three layers: stamper routing, declared soul, observed behavioral pattern) is a separate effort, see [../KNOWITSLLM.md](../KNOWITSLLM.md). The Word's part is only that declarations stay uniform across cognitions while the substrate records who wrote each.
- The Word is generative. Realities are not spoken into being once; the language they are spoken in keeps growing.

## roleFlow becomes declarative

Dynamic role selection (which role applies right now) and modifier composition (emotional, alert, temporary modes stacking) stay, but move from a half-imperative JavaScript array into the same query language as can-see:

```
When the space is a court and the court is in session, use the judge role.
Otherwise use the court-watcher role.
When the time since the last moment is over 60 seconds, add the bored modifier.
```

Because roles and roleflows are both declarations, the substrate can compose them, inherit them, layer a space's roleflow over a being's, and let a being introspect its own role logic. roleFlow is the right first target to test the language against; the existing `when` system is already a small DSL pointing at this.

## The universal interface

The Word is not only for spawning. It is the one medium for every interaction with the substrate:

- **Declare** structure: a music-room is a space.
- **Define** roles and their capabilities, and flow rules.
- **Spawn** instances: spawn a music-room called "lounge".
- **Install** packs: install the music-room pack from tabors-site.
- **Grant** roles: grant Alice the dj role for lounge.
- **Act:** queue "Sunflower" to lounge's playlist.
- **Modify** behavior live: for lounge, listeners can vote to skip; when over half vote, skip the song.
- **Query** state: find all songs queued more than 5 times this week.

Two consequences:

- **A Word program is a seed.** A `.word` file with declarations plus an instantiation command is a plantable seed. The Word is both the language for declaring AND the format for distributable substrate content (resource packs). This unifies what are today separate mechanisms: JavaScript for type definitions, manifests for packs, API calls for spawning, code changes for flow, can-see for queries.
- **Word is to TreeOS what natural language is to human community:** the medium through which everything gets done. It applies everywhere intention, specification, or interaction happens. It does not apply in the implementation layers underneath (parser, evaluator, storage, network), which stay in the host language.

One more property worth naming: a Word program is both static definition and active invocation, depending on the statements it holds. "A music-room is a space" defines; "spawn a music-room called lounge" invokes; same syntax, same file. That is what dissolves the line between a resource pack and an API call.

## The escape hatch

Pure declaration cannot express real computation: parsing input, calling an external service, running an algorithm or an ML model. For that, a declaration includes a bounded compute block that calls into the host language and feeds the result back into the native declaration.

```
An analyze act computes its result with: js runAnalysis(dataset.content).
```

This mirrors stored procedures in SQL or built-ins in Datalog: a declarative core with controlled escape hatches. The Word covers substrate-shaped logic; the host language (JavaScript, for now) covers everything that is not substrate-shaped, plus the substrate's own machinery (the stamper, the fold engine, the network). Put crisply: The Word covers substrate content; the host covers substrate machinery.

## Grammar patterns

Consistent across the worked examples in 0.md (music-room, journal, study-group, document, garden, shop, tic-tac-toe, house, a whole reality):

- `A X is a Y.` declares what kind of thing exists (structure).
- `X can see / can do / can summon ...` declares a role's capabilities, third person.
- `When X happens ...` / `If X is true ...` declares a flow rule; the consequence follows.
- `X extends Y.` is role inheritance: the new role includes the parent plus its own additions.
- `X is a role for Y.` binds a role to the kind of space it applies in.
- Imperative verbs in act bodies (queue, play, notify, record) are the substrate operations the act performs.
- Numbers and ranges use natural notation (0 to 100, 7 days, 3 of the same mark), not symbolic operators.

Constrained enough to parse, natural enough to read.

## How it gets built

- **The AST is native data.** Parse the word-shaped surface into plain objects with explicit type fields. The substrate operates on those objects directly; there is no separate compiler.
- **Evaluator first, then parser.** Hand-build ASTs and write the evaluator to validate the semantics, then add the parser so syntax produces identical ASTs.
- **Parser:** start with hand-written recursive descent. The constrained grammar is grounded in linguistic structure (a small subset of English), so it leans on well-understood parsing territory rather than rigid lexing alone.
- **Migrate incrementally.** New roles, acts, and flows in The Word coexist with old JavaScript handlers; each migration deletes the corresponding JavaScript.
- **Make it federable.** ASTs serialize as JSON, store in localStore, publish through a Store, draw into other realities.

Effort: a usable first version in months; maturity (grammar that handles what people actually want to write) over a year or two. The discipline is keeping the language small and using the escape hatch for everything that is not substrate-shaped, rather than letting it grow into another general-purpose language.

## Honest constraints and open edges

- The evaluator runs constantly (perception and authorization every moment); it needs caching, incremental evaluation, and indexing.
- Word-shaped parsing is harder than rigid syntax; it commits to a lightweight constrained-grammar parser, not just a lexer.
- The compute escape hatch is powerful and is a security and complexity surface; host calls from a declaration need sandboxing.
- Grammar gaps flagged as candidates, not settled (see the Status note under "Primitives are parts of speech"): pronouns and variable binding for multi-clause declarations, quantifiers, articles (definite versus indefinite), and adverb-style act modifiers. These are the same extended-grammar items still to be approved; add them only as real programs demand.

## Evolved away from (discarded as the idea matured)

- **The naming search.** 0.md spends a long stretch weighing Worldscript, Loom, Treescript, Charter, and IBP Markup, and leans toward Loom. That is superseded. Once the commitment to word-shaped declarations is made, "The Word" is not a poetic choice but a structural fact: a literal language of words occupying the Logos position. Name settled: The Word, `.word`.
- **The curly-brace syntactic form.** The early sketches use code-block syntax (`role dj for music-room { can-see: spaces where parent = music-room }`). That is the larval form. The evolved surface is word-shaped prose ("A dj can see everything in the music-room."). The brace form is still useful for picturing the AST and semantics, but it is not the language's face.
- **"Syntactically declarative is enough."** Explicitly outgrown. Syntactic declaration is halfway. The payoff properties (LLM-native authorship, self-description in English, modification-as-conversation, translation) only emerge when the language is actually word-shaped.
