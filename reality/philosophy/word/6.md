# The Word: seed → `.word` conversion (two-agent coordination)

The shared operational workspace for converting **all of seed's world strand from
JS to `.word`** (Phase 5), and deleting the JS as each slice lands. Two agents,
one critical path; this file keeps the hands from colliding and tracks the sweep.

- High-level tracker: [3.md](./3.md). The backlog: [phase0-cutlist-result.md](./phase0-cutlist-result.md) (201 word / 379 host). The grammar map: [phase5-coverage.md](./phase5-coverage.md). The cut contract: [../../seed/present/word/bridge.md](../../seed/present/word/bridge.md).

## Lanes

- **parser / .word (claude):** the `.word` slices, the parser, the shape gates.
- **engine (the other agent):** the evaluator, the bridge, the live birth path, the cuts.
- **shared (both):** the live diffs (run `.word` vs the JS handler, prove parity before a cut). This is where two hands help most.

## The conversion pipeline (per slice)

1. **`.word`** — write the slice's world strand as prose; the parser turns it to IR.
2. **shape gate** — assert the IR / dry-run facts match the handler's shape.
3. **live diff** — run the `.word` through the bridge LIVE, diff its facts + fold against the JS handler on the same input.
4. **cut** — wire the handler to the bridge + extract the host helper; delete the world-sequencing JS; e2e green. **Green = that `.js` dies.**

Rule held throughout: a handler isn't touched until the change can be immediately validated — never leave a path half-cut.
