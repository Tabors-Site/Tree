# Word ↔ system naming mismatches (reformat-later log)

Where the system's names or shapes don't match the Word's clean clause, and the
Word has to bridge across the gap. **We do not conform the system now** (the facts
are canon-frozen, renames break the chain), we just EXPOSE the mismatch here. At
the end we do one reconciliation pass so the system's field names ARE the Word's
clause properties (`by` in the Word is `by` in the fact too), and word and system
read as one thing. Word reads cleanly meanwhile; the bridging lives in the
evaluator / parser until then.

Both agents add rows as they hit them.

## The act / fact shape

| Word clause property | system field today | note |
|---|---|---|
| `by` (the actor Name, rule 9) | `beingId` | the fact's actor field is `beingId`. The Word calls it `by`. And the Word SPLITS `by` (the Name, the signer) from `through` (the vessel being, rule 9) — `beingId` conflates them into one. |
| `through` (the vessel being) | (none) | the by/through split has no field in the fact yet; the vessel is folded into `beingId`. |
| `of` (the object acted on) | `target: { kind, id }` | the Word names the acted-on `of`; the fact uses `target`. |
| `to` (the receiver, rule 17) | (none) | new in the Word (`I give the drum to Claude`); the fact has no receiver field. The evaluator now carries `to`, but the fact schema has nowhere settled for it. |
| `op` (the operation within a verb) | `action` (sealed); `act` (in flight) | the Word clause is `verb` + `op`. The fact is `verb` + `action` (canon-frozen). The `act → action` at-the-seal split is already known; `op` is the Word's third name for the same slot. |
| (the args, typed) | `params` (a bag) | the Word clause carries typed objects, not a `params` blob; today the evaluator stuffs them into `params`. |
| a being by its **proper name** (`Cherub`, `Arrival`) | a being **id** (content hash) | the Word names beings by proper noun (rule 19); the system keys them by id. cherub.word's `parentBeingId`/`through`/lineage carry `Cherub`/`Arrival`; the live op handlers need ids. Bridged in the evaluator via `ctx.beings` (proper-name → id, scoped so ordinary strings pass through). End-reformat: a name registry so `.word` stays in proper names and the system resolves ids. Surfaced landing the cherub live diff. |
| the **actor** (`by` = a Name) for authorize | `identity: { beingId, name, nameId }` + the I_AM short-circuit (`name === "i-am"`) | authorize gates on an `identity` object, not the `by` Name directly. For "by I_AM through Cherub", the run must carry `identity.name === "i-am"` (the bootstrap axiom short-circuit) with `beingId` = the vessel; the summoned-being identity alone is denied for the privileged acts. The Word's `by`/`through` and the system's `identity`/authorize are the same idea under two shapes. |

## What stays host (rule 3, NOT a mismatch to fix)

`_inOp` / `_opCount` (one-op-per-moment), `stamp` / `fold` / `seal`, `prev` / hash
/ `seq` / `branch` (the chain envelope). These are the HOW; the Word never names
them, and they should NOT be reconciled into Word shape. They're listed here only
so nobody mistakes them for mismatches.

## The reconciliation pass (end of Phase 5, NOT now)

When the world strand is `.word` and the conformance suite is frozen, rename the
fact/op fields to the Word's clause properties (`by`, `through`, `of`, `to`, `op`),
or settle that the fact's CONTENT simply IS the act-AST (1.md, the fact-chain
resolution: the act node is the fact content, one schema). Either way the goal is:
the name you write in the Word is the name stored. Until then, bridge in the
evaluator, expose here, break nothing.
